## Зачем вообще ещё один ИИ-чат

Рынок переполнен обёртками над чужими API. Мы строим связку **модель + продукт + tools**:

- четыре модели в одном интерфейсе с понятной ценой за ответ;
- веб-поиск (**Xelity Search**) и погода (**Xelity Weather**) прямо в чате;
- режим кодинга с файловой песочницей и превью сайта;
- публичное API: Chat Completions (OpenAI-стиль) и Messages (Anthropic-стиль).

Сайт: [https://xelity.ru](https://xelity.ru)  
API: `https://api.xelity.ru`

Режим **«Рассуждения»** удваивает стоимость ответа (два шага генерации).

## Что умеет продукт (веб-чат)

Из того, чем реально пользуются каждый день:

1. **Чаты и папки** — история, автоназвания, поиск по переписке, сортировка.
2. **Выбор модели** на лету — без смены аккаунта.
3. **Xelity Search** — поиск с источниками и опционально картинками.
4. **Xelity Weather** — прогноз по месту (в чате и через API).
5. **Кодинг** — песочница файлов, правки через tools, проверка сборки и превью.
6. **Тема / язык / размер UI** — мелочь, но без неё долго работать неприятно.

Технически фронт — React + Vite, бэкенд чата и API — Node на VPS, синхронизация профиля — Firebase. Для пользователя это просто «открыл чат и работаешь».

## Публичное API

Ключи вида `xel_…` создаются в кабинете `/account/api`. Баланс — в виртуальных USD (пополнение через админку продукта).

Основные эндпоинты:

- `GET /v1/models` — список моделей и цены;
- `POST /v1/chat/completions` — OpenAI-совместимый chat;
- `POST /v1/messages` — Anthropic-совместимый Messages;
- `POST /v1/search` — Xelity Search;
- `POST /v1/weather` — Xelity Weather.

Пример минимального вызова:

```bash
curl https://api.xelity.ru/v1/chat/completions \
  -H "Authorization: Bearer xel_YOUR_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "xlaude-mini-k1",
    "messages": [
      {"role": "user", "content": "Кратко: что такое Xlaude?"}
    ]
  }'
```

ID моделей в API совпадают с продуктом: `xlaude-mini-k1`, `xlaude-pro-k1`, `xlaude-mini-k2`, `xlaude-pro-k2`.

Ориентиры цен (виртуальные USD за 1M токенов):

| Модель | Input | Output |
| --- | --- | --- |
| Mini K1 | $0.40 | $1.20 |
| Pro K1 | $0.80 | $2.40 |
| Mini K2 | $1.60 | $4.80 |
| Pro K2 | $3.20 | $9.60 |

Search — от $0.01 за вызов, Weather — $0.005. При нулевом балансе — HTTP 402.

В веб-чате tools крутятся на стороне продукта. В публичном chat API tools сервером не исполняются: Search и Weather вызывайте отдельными `POST`.

## Как мы думаем о продукте

Несколько принципов, которые реально влияют на UX:

1. **Цена ответа должна быть прозрачной.** Кредиты за модель видны сразу; нет сюрприза «сообщение съело половину лимита непонятно почему».
2. **Tools — часть ответа, а не отдельный «режим бога».** Поиск и погода встраиваются в диалог; кодинг — когда нужен сайт/код, а не стена извинительного текста.
3. **API должно быть скучным в хорошем смысле.** Совместимость с привычными клиентами важнее собственного «уникального» протокола.
4. **Оценки — с оговоркой.** Лучше честно написать «внутренний прогон», чем притворяться независимым лидербордом.
